# #s3

## Overview
This channel is primarily used for the development of BST's S3 VTOL aircraft - a long-endurance drone with vertical takeoff/landing capability. The S3 is designed as a hybrid fixed-wing/quadcopter aircraft with tilting rotors, targeting 2-3 hour endurance with automatic flight operations.

## Key Decisions

**Battery Configuration (2020)**
- Selected 6S10P 18650 cell configuration using LG HG2/VTC6 cells over LiPo packs
- Established 120-second VTOL time budget (60s takeoff + 60s landing)
- Later upgraded to 12S configuration for improved performance
- Current configuration: 2x 540Wh packs (1080Wh total) for S3

**Motor Selection (2023-2024)**
- Chose Vertiq 81-08 G2 motors with integrated ESCs ($700 each) for rear rotors
- Selected KDE motors as backup due to Vertiq supply delays
- Rejected T-motor options due to NDAA/China restrictions
- Up to 6 Vertiq motors available from vendor currently; company has 3 unused + 2 in use on mass model (as of April 22, 2026)
- **Deferred motor platform decision** (April 22, 2026): Decided to continue with Vertiq for now; ground short issue resolved with simple insulating kapton film spacer; revisit motor choice at group meeting

**Design Configuration**
- Wing extension: 16" added to handle VTOL weight
- Tilting rotor design: 10-15 degree tilt angle for efficiency
- Battery placement: Moved from rear to front fuselage for CG balance
- Color scheme: Purple and gold selected for first prototype

**Control Architecture (April 22, 2026)**
- **MAJOR DECISION**: Switching from DroneCAN to PWM control for first S3 aircraft
- Rationale: Too many delays waiting to validate DroneCAN with issues on S1-22; moving to PWM allows faster progress
- Ailerons already swapped to PWM (even for last flight); remaining items are DroneCAN-related
- Joshua Fromm estimate: Less than 2 hours to complete PWM swap
- DroneCAN will remain primary for future/parallel testing on S1-22

**Servo Selection (April 2026)**
- Replaced servos with new units supporting up to 333Hz update frequency
- Left aileron on address 1, right aileron on address 4
- New servos have slightly different down deflection due to spline offset - acceptable given fixed-length linkages
- Caution: Manual upward aileron movement can overextend wipers, reducing down deflection if not caught

**Flight Parameters (April 2026)**
- Climbout throttle set to 75% (starting value, reduces if overspeeding or over climbing)
- Weight for mass model testing: 15.1kg
- Estimated full weight with payloads: 16kg
- Speed scaling: 19m/s (S2) → 21m/s (S3 at 16kg) based on mass difference

**Photogrammetry Payload Camera Triggering (April 17, 2026)**
- Issue identified: Trigger tests on preflight page trigger both cameras (Sony and FLIR) simultaneously, but from surfaces page cameras can be triggered independently
- Requires investigation into preflight page trigger configuration
- DroneCAN termination configuration: Payload has termination at end of chain but is connected to last actuator board which is also terminated - needs review
- Random Sony A5100 trigger events may be bus-related but appear normal/expected to prevent timeout conditions

**GNSS Module Selection (April 21-22, 2026)**
- **New GNSS boards identified as problematic**: Units incorrect for GPS course (degrees vs radians mismatch with controller expectations)
- **Decision to revert to older GNSS boards**: Swap to S2-style GNSS for initial S3 testing to enable progress on mechanical/wiring/voltage validation
- Joshua Fromm estimate: <2 hours to swap, substantially less if voltage compatibility confirmed (Jack Elston checking +24v vs +5v compatibility)
- Older GNSS boards (v06 style) include DEG_TO_RAD conversion; new v08 boards missing this conversion
- RTK-related PDOP bounce issue identified as root cause of pitch estimation problems; software fix would be "fairly involved" on AP side
- Fallback strategy: Use older proven GNSS while Sam Hild debugs RTK issues separately

- Joshua Fromm adjusted Vertiq motor setting: Changed `Volts Limit Starting Voltage` from 50.0V to 0.0V
- Rationale: Limits regen to absolute minimum during braking while maintaining braking function
- Planned test: Spin up front right motor to validate fix works

- Reference: IQMotion regeneration safety systems documentation
