## Executive Summary
**Note: This document is NOT from Black Swift Technologies.** This is a Phase II interim report template/sample from Coney Island Incorporated (Brooklyn, NY) for a NASA SBIR contract on "Network Implementations for Real Time Communication and Data Transfer." The document describes development of OPC (OLE for Process Control) based Quality of Service network protocols for networked intelligent devices such as sensors and controllers.

## Technical Approach
The contractor proposed developing an OPC-based QoS network protocol with the following approach:
- Client/server architecture using OLE/COM technology
- OPC server design with components: server objects, group objects, and item objects
- Quality of Service (QoS) implementation focusing on bandwidth management and traffic control
- Two QoS categories: Resource Reservation and Prioritization
- Development stages: in-proc servers, local servers, and remote servers
- Object-Oriented Design (OOD) methodology for implementation
- Client-side communication traffic monitoring, QoS management, and bandwidth adaptation

## Products & Capabilities Described

### Electronic Handbooks (EHBs)
- Internet-based tools for paperless documentation and management of complex distributed processes
- Support for Grants/Contracts Management
- Guided workflows for multiple actors/roles communicating via Internet
- Applications: NASA SBIR Program, Department of Justice Bulletproof Vests Partnership (BVP) Program

### OPC (OLE for Process Control) Network
- Enables client applications to control and manage communication among different terminal devices (OPC servers)
- Supports connection of diverse devices: PLCs, sensors, controllers from multiple manufacturers
- Implements QoS to support fast communication and data transfer
- Hardware-independent, plug-and-play operation
