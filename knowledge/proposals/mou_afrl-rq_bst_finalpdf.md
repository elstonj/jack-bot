## Executive Summary
This MOU documents AFRL/RQQC's interest in and commitment to partnering with Black Swift Technologies on a Phase II SBIR project to develop artificial intelligence algorithms for preventative maintenance, fault detection, and flight envelope monitoring on autonomous UAS platforms. The work will adapt BST's AI tools (previously matured under NASA SBIR efforts) to the Cloud Cap Technology Piccolo autopilot, with intended application to AFRL's Group 3 fixed-wing and hybrid VTOL UAS platforms as well as operational DoD systems.

## Technical Approach
- **Primary Goal**: Apply Black Swift Technologies' mature AI tools to the Cloud Cap Technology Piccolo autopilot for preventative maintenance tracking and envelope monitoring
- **Key Technical Components**:
  - Development of a web-based software portal for automatic anomaly detection on UAS subsystems
  - Processing of flight logs from Group 1 and Group 2 UAS platforms
  - Machine learning network architecture (identified during Phase I) for fault detection with focus on minimizing false positives and false negatives
  - Cloud-based storage and security compliant with AFRL requirements
  - Real-time performance envelope evaluation and flight demonstrations
  - Open API development for BST and AFRL to enable future algorithm development
  - GAN (Generative Adversarial Network) implementation on BST and AFRL-supplied data
- **Platform Integration**: Certification of S2 UAS for AFRL use and development of framework for performance envelope tracking

### S2 UAS
- Black Swift's small unmanned aircraft system
- To be certified for AFRL use under this project
- Will undergo flight testing and demonstration of real-time performance envelope evaluation
- Proposed for use in continued preventative maintenance and performance envelope analysis

### AI/ML Algorithms for Predictive Maintenance
- Preventative maintenance detection algorithms adapted from prior NASA SBIR work
- Fault detection capabilities for autonomous UAS systems
- Online envelope monitoring for real-time platform readiness assessment
- GAN implementation for improved detection over current approaches
- Automated anomaly detection with minimized false positives/negatives

### Cloud-Based Portal/Server Solution
- Web-based portal for anomaly detection on various UAS subsystems
- Ingestion of flight logs and maintenance/failure records
- Cloud-based storage and security compliant with AFRL requirements

## Use Cases & Applications

### Primary Military Applications
- **Autonomous Squadron Readiness**: Improved maintenance tracking for AFRL/RQQC-owned Group 3 fixed-wing and hybrid VTOL UAS platforms
- **Operational DoD UAS Platforms**: Direct application to fielded DoD systems
- **Attritable Platforms**: Real-time flight envelope performance monitoring to inform commander decision-making on mission success vs. platform loss
- **Manned-Unmanned Teaming**: Envelope monitoring and estimation to augment autonomous contingency planning, reducing pilot workload
- **Special Operations**: Previous support to air-launched UAS programs for NOAA; potential transition to AFRL/RQQ SUAS assets through special operations community

### Specific Target Programs for Phase III Transition
- **ALOBO Program** (Air Launched Oboard Operations): Primary transition target within AFRL/RQQC; BST has previous and current experience supporting air-launched UAS programs for NOAA
- **Skyborg Vanguard Program**: Would benefit significantly from real-time fault detection, envelope estimation, and maintenance identification systems

### Operational Benefits
- Reduced manpower burden for operation and maintenance of fielded systems
- Enhanced flight platform readiness
- Real-time information on platform readiness, capability, and survivability for both commanders and autonomous decision-making algorithms
- Increased mission effectiveness and squadron readiness
