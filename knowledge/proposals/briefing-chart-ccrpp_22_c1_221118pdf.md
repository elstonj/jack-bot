# A Ruggedized UAS for Scientific Data Gathering in Harsh Environments

## Document Metadata
- Type: SBIR Phase III briefing chart
- Client/Agency: NASA
- Program/Solicitation: NASA SBIR (2016 SBIR Phase II-E volcano topic)
- Date: January 18, 2022
- BST Products/Systems Referenced: S2-VTOL, SwiftCore (firmware), SwiftTab (app)
- Key Personnel: Jack Elston (PI, Black Swift Technologies)

## Executive Summary
Building on Phase II-E work conducting volcano monitoring with small UAS, this Phase III effort proposes to automate mission capabilities for harsh environment operations. The focus is implementing intelligent flight planning for strong winds, improving the user interface for payload management, automating terrain/communication-aware mission planning, and improving manufacturability of the S2-VTOL platform to enable routine worldwide deployment in difficult weather conditions.

## Technical Approach
Four main technical objectives:

1. **Wind-Aware Intelligent Flight Planning**: Implement real-time environmental assessment and predictive modeling to inform dynamic flight path adjustments that minimize energy usage. Changes will be integrated into SwiftCore firmware and demonstrated in flight testing. This addresses the current manual requirement for constant wind monitoring and battery margin calculations.

2. **User Interface Improvements**: Enhance SwiftTab app to provide flexible telemetry display and payload control. A common operating picture is needed to accommodate interchangeable payloads and support scientific data management for various mission types.

3. **Automated Terrain & Communications Planning**: Extend mission planning and flight control automation to account for terrain navigation and communication constraints, similar to wind-aware planning. Addresses limitations of manual, months-long mission preparation currently required.

4. **S2-VTOL Manufacturability Improvements**: Refine the S2-VTOL design to simplify construction and reduce cost while maintaining the key capability of launching/landing in strong wind conditions—currently identified as a limiting factor for small UAS operations.

## Products & Capabilities Described

**S2-VTOL**
- Prototype vertical takeoff/landing small UAS platform
- Key advantage: ability to launch and land nearly anywhere during strong wind events
- Proposed improvements: simplified manufacturing, reduced price
- Addresses a current limiting factor for small UAS operations in harsh conditions

**SwiftCore (Firmware)**
- Flight control and mission execution firmware
- Proposed enhancements: wind-aware flight planning, real-time environmental assessment, predictive modeling, automated mission adjustments

**SwiftTab (App)**
- Ground station/user interface application
- Current limitation: not flexible enough for multi-payload operations
- Proposed improvements: common operating picture, flexible telemetry display, payload control interface

## Use Cases & Applications

**Primary:**
- Volcano monitoring (specifically referenced as Phase II-E application)
- Operations in convective storms
- Mountainous terrain flight

**Implied broader applications:**
- Routine scientific data gathering in harsh/difficult weather conditions
- Worldwide deployment missions
- Missions with challenging terrain navigation and communication constraints

## Notable Details

- **Phase II-E Context**: Previous phase achieved "only visual observations of the target volcano during that calendar year," demonstrating mission success but revealing operational complexity
- **Current Operational Burden**: Mission planning currently requires months of preparation; flight execution requires a "very knowledgeable and well trained team" with frequent real-time wind monitoring and custom software calculations
- **Automation Goal**: Intent is to make routine, worldwide harsh-environment operations viable by automating capabilities currently requiring expert manual management
- **VTOL Advantage**: Strong wind launch/landing capability identified as competitive advantage and current operational limiting factor for small UAS
- **Interchangeable Payloads**: System architecture supports multiple scientific payload configurations
