## Executive Summary
This MOU documents the Air Force Research Laboratory's (AFRL/RQQC) commitment to collaborate with Black Swift Technologies on an SBIR Phase II project focused on developing artificial intelligence algorithms for preventative maintenance, fault detection, and online envelope monitoring of unmanned aerial systems (UAS). The effort aims to transition autonomous UAS technologies to warfighters by improving flight platform readiness and reducing operational manpower burden through AI-driven predictive maintenance and real-time monitoring capabilities.

## Defense Mission Need
AFRL/RQQC identified a critical national defense-related mission need:
- **Core Problem**: Enabling autonomous UAS deployment at scale requires improved systems and processes for flight platform readiness and maintenance tracking
- **AI Application**: Leveraging artificial intelligence for predictive maintenance to significantly reduce the manpower burden for operating and maintaining fielded systems
- **Key Benefits**:
  - Real-time platform readiness, capability, and survivability information to inform commanders and autonomous decision-making algorithms
  - For attritable platforms: reduced flight envelope performance data enables better mission success vs. platform loss tradeoffs
  - For manned-unmanned teaming: envelope monitoring and estimation augment autonomous contingency planning and reduce pilot workload
- **Target Platforms**: AFRL/RQQC owns and operates several Group 3 fixed-wing and hybrid VTOL UAS platforms that will directly benefit from improved maintenance tracking
- **Mission Impact**: Increased UAS squadron readiness, reduced maintenance and inspection manpower, increased mission effectiveness for autonomous UAS platforms

## Technical Approach

### Primary Technical Focus
BST will develop and mature AI tools previously developed under prior SBIR efforts with NASA, applying them to the Cloud Cap Technology Piccolo autopilot for:
- Preventative maintenance tracking on AFRL surrogate UAS
- Integration with operational DoD UAS platforms
- Onboard monitoring and envelope tracking to increase capabilities of open autonomy architectures

### Work Plan Details (Non-Proprietary)
The Phase II effort will deliver a software portal for automatic anomaly detection on UAS subsystems through flight log processing:

1. **Machine Learning Infrastructure Enhancement**:
   - Modify BST's current log server to incorporate new machine learning network architecture identified during Phase I
   - Architecture provides significant advantages over current fault detection approaches
   - Web-based portal demonstrating algorithm performance comparison to previous iterations
   - Focus on minimizing false positives and false negatives in anomaly detection

2. **Flight Log Processing & Validation**:
   - Ingest AFRL-supplied log files, maintenance records, and failure records
   - Automated ingestion method into existing BST framework
   - Validation through display of analyzed data on BST cloud-based server solution
   - AFRL technical monitor and team provided login access for review

3. **Cloud Compliance & Security**:
   - Cloud-based storage and security methods compliant with AFRL requirements
   - Implementation and validation details to be provided in status reports

4. **S2 UAS Certification & Documentation**:
   - Develop necessary documentation and certification for S2 UAS approval for preventative maintenance and performance envelope analysis work
   - Open API development for BST and AFRL use in future algorithm development
   - Flight demonstration of real-time performance envelope evaluation

## Products & Capabilities Described

### S2 UAS
- **What it is**: Small unmanned aerial system operated by BST
- **Proposed Use**: Flight testing and real-time envelope monitoring demonstration; approved platform for continued preventative maintenance and performance envelope analysis work
- **Capabilities**: Will undergo certification and documentation for ongoing use in algorithm development and validation

### Cloud-Based Server Solution
- **What it is**: BST's existing cloud infrastructure adapted for this project
- **Proposed Use**: Host web-based portal for anomaly detection algorithm demonstration; store and process flight logs; provide AFRL stakeholders with secure login access for data review
- **Requirements**: Must be compliant with AFRL security and storage requirements

### Machine Learning Algorithm Architecture
- **What it is**: New network architecture identified during Phase I of this SBIR effort
- **Proposed Use**: Automatic anomaly detection in UAS subsystems via flight log analysis; fault detection with reduced false positives/negatives
- **Enhancement**: Represents significant improvement over previous fault detection approaches
- **Deliverable**: Web-based portal demonstrating algorithm performance

### Open API
- **What it is**: Software interface for algorithm integration
- **Proposed Use**: Enable both BST and AFRL to develop future algorithms; facilitate integration with other systems and platforms
- **Scope**: To be developed as part of S2 certification documentation

## Use Cases & Applications

### Primary Use Cases
1. **Preventative Maintenance Tracking**: Automatic detection of anomalies in Group 1-3 UAS flight data to enable proactive maintenance scheduling and reduce unscheduled downtime

2. **Real-Time Envelope Monitoring & Estimation**: On-platform monitoring of flight envelope performance to inform:
   - Commander decision-making on mission risk vs. platform loss tradeoffs
   - Autonomous vehicle contingency planning algorithms
   - Pilot workload reduction in manned-unmanned teaming scenarios

3. **Attritable Platform Operations**: Reduced flight envelope performance detection enables better operational decisions for platforms designed for single-use or high-loss scenarios

### Target Platforms & Programs
- **Surrogate Testing**: AFRL/RQQC Group 3 fixed-wing and hybrid VTOL UAS platforms for risk reduction and maturation
- **Phase III Transition Targets**:
  - **ALOBO (Air Launched Offboard Operations) Program**: AFRL/RQQC program with existing BST relationship (BST currently supports air-launched UAS programs for NOAA)
  - **Skyborg Vanguard Program**: Would significantly benefit from real-time fault detection, envelope estimation, and maintenance identification systems
  - **AFRL SUAS (Small UAS) Assets**: Potential integration through Phase III SBIR
  - **Special Operations Community**: End-use customers for air-launched small UAS

## Phase III Transition Strategy
AFRL/RQQC commits to considering transitioning successful Phase II innovations through multiple pathways:

### Transition Mechanisms
1. **Phase III Contracts** (preferred under SBIR regulations):
   - Sole source award to Black Swift Technologies via SBIR Phase III special acquisition preference
   - Potential use of sole source authority under Section 8(a), HUBZone, WOSB, or SDVOSB programs

2. **Alternative Instruments**:
   - Non-procurement instruments (grants, cooperative agreements, other transaction agreements) with or without follow-on production contracts
   - Assisted acquisition agreements

### Scope of Phase III Activities
Potential Phase III contract may cover:
- Further development, testing, and evaluation of artificial intelligence algorithms
- Production and delivery of avionics hardware
- Integration of avionics hardware on relevant UAS platforms

### Stakeholder Engagement Strategy
AFRL/RQQC will:
- Engage internal and external partners to coordinate interest
- Gather stakeholders from across DoD UAS community in support of BST AI algorithms
- Work with end-user customers (special operations community) for transition to operational assets
- Serve as integration conduit for novel autonomy/AI components into military UAS fleet
