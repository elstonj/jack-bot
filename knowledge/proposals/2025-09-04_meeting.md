## Executive Summary
BST presented Phase I progress on a Navy SBIR project to develop an unmanned aerial vehicle capable of being launched from sonobuoys for anti-submarine warfare operations. The presentation covers magnetometer and acoustic sensor integration challenges, test results showing noise and heading error issues, and sensor requirements for upcoming development milestones due by October 14, 2025.

## Technical Approach

**Platform Integration:**
- Integrating magnetometer and acoustic hydrophone sensors onto BST's S0 air-deployed platform
- Modifying S0 to enable sonobuoy launch capability
- Direct sensor-to-autopilot integration planned to replace current Sensor Control Board (SCB) integration

**Magnetometer Testing & Challenges:**
- System time jitter issues requiring time-sync solutions for real-time processing
- Heading error of 120-150 nanoTesla (nT) caused by magnetic components on platform; mitigatable through calibration and removal of magnetic materials
- Platform noise levels of 250-600 picoTesla/√Hz (10-30x requirements)
  - Electric motor generating ~100 nT of noise requiring magnetic shielding
  - Mitigation strategy: magnetic shielding of motor components
- Column naming/data formatting issues in initial test data

**Acoustic Sensor (Hydrophone):**
- Omnidirectional hydrophone deployment system
- Operating frequency range: ~1 Hz to 25 kHz (threshold 0.01 Hz–2.5 kHz; objective 0.001 Hz–25 kHz; can achieve ~1 Hz minimum)
- Low-noise design targeting sea state zero or better across operating bandwidth
- Data whitening/shaping for bandwidth-reduced uplink

## Products & Capabilities Described

**S0 Platform:**
- Air-deployed UAS baseline for ASW sensor integration
- Significantly smaller than initial test platform used for magnetometer validation
- Current size/weight constraints challenging for multi-sensor payload integration
- Team exploring need for "significantly modified airframe" to accommodate magnetometer, hydrophone, and other required sensors within SWaP limits

**Magnetometer Sensor:**
- Integration with autopilot system
- Requires calibration flights post-integration
- Magnetic shielding requirements for motor noise mitigation

**Acoustic Hydrophone Sensor:**
- Threshold operating life: 60 minutes; Objective: 70 minutes
- Threshold max operating depth: 200 ft (60.96 m); Objective: 400 ft (121.92 m)
- Deployment time threshold: 120 s; Objective: 60 s
- Omnidirectional (threshold requirement); direction-finding not feasible with current design
- Calibration accuracy: +/-2 dB likely achievable; +/-1 dB potentially possible pending testing; +/-3 dB confirmed achievable
- Sensor noise equivalent designed to approach sea state zero performance

## Use Cases & Applications

**Primary Mission:** Anti-submarine warfare (ASW) cued search, detection, tracking, and classification using sonobuoy-launched UAS

**Secondary Application (mentioned as context):** Wildfire operations with real-time wind and turbulence data gathering for predictive model assimilation and distribution to firefighting teams

## Key Results (Magnetometer Testing)

**Issues Identified:**
1. System time jitter requiring synchronization improvements
2. Heading errors of 120-150 nT from platform magnetic field
3. Platform electromagnetic noise 10-30x over requirements (250-600 pT/√Hz vs. ~25-60 pT/√Hz target)
4. Motor noise contribution: ~100 nT

**Mitigation Strategies Proposed:**
- Magnetic shielding of electric motor
- Removal of magnetic components where possible
- Calibration flights post-integration
- SCB elimination in favor of direct autopilot integration

## Notable Details

**Major Challenges:**
- S0 platform significantly smaller than test vehicle; scaling concerns for sensor integration
- Platform noise levels substantially exceed requirements; motor shielding critical path item
- Acoustic sensor cable/reel requirements for 400 ft depth operations present mechanical/packaging challenges
- Hydrophone MCU and analog-to-digital conversion appear to be Navy/customer responsibility ("That is not our scope, do you have an MCU perhaps...")
