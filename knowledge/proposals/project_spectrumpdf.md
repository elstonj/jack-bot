## Executive Summary
This document is a NIST 800-171 cybersecurity self-assessment tool hosted on projectspectrum.io. It provides a structured checklist for organizations to evaluate their compliance with NIST 800-171 security requirements, specifically focusing on the "Access Control" category of controls.

## Technical Approach
The document presents a questionnaire-based assessment methodology organized around 21 specific access control requirements from NIST 800-171. Each requirement is broken down into granular sub-questions with Yes/No/Not Applicable response options, allowing organizations to systematically evaluate their current state against federal cybersecurity standards.

## Key Assessment Areas (Access Control)
1. User, process, and device access authorization and limitation
2. Transaction and function authorization
3. CUI information flow control
4. Separation of duties
5. Least privilege enforcement
6. Non-privileged account usage for non-security functions
7. Privileged function prevention and audit logging
8. Failed logon attempt limitation
9. Privacy and security notices for CUI
10. Session lock mechanisms with pattern-hiding displays
11. Automatic session termination
12. Remote access monitoring and control
13. Managed access control points for remote access
14. Remote privileged command execution authorization
15. Wireless access authorization
16. Wireless authentication and encryption
17. Mobile device connection control
18. Mobile device CUI encryption
19. External information system connection verification and control
20. Portable storage device use limitations
21. FCI/CUI protection on publicly accessible systems
