# Extensible Multi-UAS Coordination Architecture

## Document Metadata
- Type: NASA SBIR Phase I Proposal
- Client/Agency: NASA
- Program/Solicitation: NASA SBIR Topic A2.02 (UAS Technology); Project Number A2.02-9831
- Date: Submitted December 27, 2016
- BST Products/Systems Referenced: SwiftCore Flight Management System (FMS), SwiftPilot autopilot, SwiftStation ground station, SwiftTab user interface, compute node (Jetson TX1-based)
- Key Personnel: Dr. Maciej Stachura (PI/CTO), Dr. Jack S. Elston (CEO/Co-founder), Prof. Eric W. Frew (Guidance and Control Expert, CU Boulder), Dr. Cory Dixon (Wireless Network and User Interface Expert)

## Executive Summary
Black Swift Technologies proposes to design and develop an extensible architecture for coordinated multi-UAS operations that enables a single operator to safely control a team of heterogeneous unmanned aircraft with reliability approaching single-aircraft missions. The core innovation is using cooperative algorithms and inter-aircraft coordination to mitigate risks of individual aircraft and subsystem failures, addressing NASA's goal of advancing from four operators per UAS to four UAS per operator. Phase I will focus on architecture design, compute node development, coordination algorithm development, wireless networking, and simulation validation.

## Technical Approach

### Core Architecture Strategy
- **Compute Node Foundation**: nVIDIA Jetson TX1 single-board computer as the backbone for each aircraft, providing computational bandwidth for cooperative algorithms, inter-aircraft messaging, mechanical/electrical interfaces for payloads/sensors, and data dissemination
- **Middleware Selection**: Core Flight Executive (cFE) selected as the messaging and coordination middleware, with open interface definitions enabling rapid addition of new aircraft and algorithms
- **Layered Design**: OSI model-based layered architecture with simple open interfaces for adding new aircraft, avionics systems, and sensing capabilities with minimal overhead
- **Hardware Interface**: Snap-on circuit board integrating Jetson TX1 with BST's SwiftPilot autopilot; designed to also interface with legacy NASA systems (e.g., Cloud Cap Piccolo)

### Inter-Aircraft Coordination Algorithms
**Two primary failure mitigation areas:**

1. **Onboard Subsystem Failures** (Lost GPS and Lost Communications):
   - Other aircraft in team use onboard camera systems with machine vision algorithms to track the malfunctioning aircraft
   - Vision-based tracking leverages OpenCV machine learning algorithms running on Jetson TX1
   - Tracked aircraft guided back to landing site by team; operator informed of position for deconfliction
   - Based on BST's existing SDK approach previously used with Gumstix SBC

2. **Inclement Weather Response** (Wind/Turbulence and Icing):
   - Multiple spatially-distributed aircraft allow early warning when adverse conditions encountered
   - Team relocates to areas where other aircraft report better conditions
   - Modified ordered upwind methods enable multi-vehicle path planning to find feasible routes through strong wind fields
   - Algorithms based on Dr. Elston's prior work on wind-aware flight planning

**Formation Flight Baseline**: Implementation of formation flight algorithm as fundamental cooperative behavior for atmospheric sampling missions; simulations will test team response to subsystem failures (lost communication, lost GPS, motor failures) during formation.

### Wireless Network Architecture
- **Generic COTS Radio Interface**: Designed to permit interfacing with different commercial radio systems (to accommodate changing mission requirements and regulations)
- **Primary Implementation**: Microhard Systems P900 radio (demonstrated ~10 km aircraft-to-aircraft range in prior SBIR work)
- **Mesh Network Capability**: P900 supports mesh networking protocol for multi-UAS scenarios; design supports scalability to 25 nodes
- **Network Functions**: Enable autonomous vehicle coordination, relay telemetry/command/control data between vehicles and operator

### User Interface Modifications
- **SwiftTab Platform**: Commercial Android tablet-based interface (reduces cost, enables end-user to select tablet meeting mission objectives)
- **Shift to Team-Based Goals**: Move from individual aircraft state commands (airspeed, altitude, waypoint) to higher-level team objectives
- **Health/Status Display**: Modified to show graphic symbology and limited but sufficient information for safer multi-UAS operation while avoiding operator overload
- **Gesture-Based Commands**: Enable one operator to efficiently control multiple aircraft
- **Algorithm Integration**: User interface will integrate with cooperative failure-mitigation algorithms to keep operator informed without information overload

## Use Cases & Applications

### Primary Mission Focus
- **Multi-UAS Cooperative Sampling**: Simultaneous sampling at spatially distributed locations; sequential sampling covering periods longer than individual aircraft endurance
- **Atmospheric Science Missions**: Volumetric sampling to determine atmospheric gradients (critical for thermodynamics, gas dispersion, transport); demonstrated with NOAA/NSF work
- **Weather Monitoring**: Aircraft operating as weather scouts providing warnings of icing and inclement weather; enables safer operations of expensive platforms (e.g., NASA Sierra)

### Historical Field Campaign Experience Referenced
- **VORTEX2** (2010): First-ever intercept of tornadic supercell thunderstorm by unmanned aircraft; demonstrated multi-aircraft coordinated operations
- **MIZOPEX** (2013, Alaska): Multi-class UAS operations (NASA SIERRA, ScanEagles, micro UAS); intensive observing campaign in Arctic; exploited unique capabilities of heterogeneous UAS
- **ARM/ERASMUS**: Arctic atmospheric sampling with two coordinated UAS in cold/harsh conditions
- **MET MAP**: Four UAS simultaneously deployed (though without aircraft-to-aircraft cooperation) for gust front and boundary layer measurements; demonstrated need for improved coordination

### Broader Applications
- Remote sensing over large areas
- Disaster response
- Agricultural support
- Collision avoidance and traffic mitigation

## Technical Objectives (Phase I)

### Overall System Goals (End State)
1. Multiple unmanned aircraft performing coordinated flight with system reliability equal to single aircraft experiments
2. Command and control of 4 unmanned aircraft by single operator via intuitive user interface
3. Inter-aircraft mesh network with 10 km range between aircraft; scalable to 25 nodes
4. Rapid implementation/modification capability for new cooperative control algorithms

### Phase I Specific Objectives
1. **Compute Node Development**: Design, build, test circuit board interfacing SwiftPilot with Jetson TX1; standalone architecture enabling future legacy system integration
2. **Coordination Algorithms**: Implement and analyze algorithms using team resources to increase robustness/reliability, targeting single-UAS mission reliability parity
3. **Wireless Radio Network**: Implementation supporting communication between heterogeneous aircraft
4. **User Interface Modifications**: Identify and prototype necessary changes for gesture-based multi-aircraft control
