## Project Notes
Project Folder:  https://drive.google.com/drive/u/1/folders/1cBn6sUtds3-xxbXZxOWfswMsQY7av97c

The overall goal of the Phase I R/R&D effort is to develop and validate a modular, layered architecture for the SwiftCore avionics system that meets NASA’s subtopic requirements for enabling aircraft autonomy. Our approach breaks the system into five functional layers:

    Multi-vehicle coordination and information gathering.
    Machine learning (ML) and machine vision–based sensing and control.
    Standard navigation (waypoint management, commanded altitude/speed, velocity vector, etc.).
    Control loop modification (tuning of PID controllers and TECS-like algorithms).
    The low-level sensor and actuator interface.

A standardized pub/sub messaging system and service discovery will allow each layer to interact with other components independently. In addition, a “safe sandbox” environment will be developed to test experimental controllers without jeopardizing flight safety, with the certified fallback system automatically taking over when unsafe commands are detected. Finally, we will demonstrate ML-enabled in-flight control loop tuning using an NDAA-compliant flight computer integrated with the SwiftCore system. Our flight test platform will be the BST S2—already developed and certified under previous NASA SBIR projects.

Milestone 1 (Month 1): Completion of a detailed architecture definition document and messaging protocol specifications.
Milestone 2 (Month 2): Successful bench testing of the newly defined messaging protocol between prototype nodes.
Milestone 3 (Month 3): Safe sandbox module prototype completed and validated in controlled bench tests.
Milestone 4 (Month 6): Demonstration HWIL test of ML-enabled control loop tuning on the BST S2 platform.
